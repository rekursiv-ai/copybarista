# Copybarista's third-party runtime dependencies -- the single declared home
# for them, kept minimal on purpose.
#
# Why this file exists: copybarista lives inside the loop monorepo, whose root
# pyproject.toml pulls the full ML stack (torch, jax, tf, pycairo, ...). Syncing
# that env just to run copybarista is wasteful and fails on CPU-only runners
# (no CUDA, no libcairo). Copybarista is itself near-stdlib, so instead of
# resolving the monorepo project it runs lean:
#
#     uv run --no-project --with-requirements <this file> python -m \
#         loop.experimental.copybarista.copybarista ...
#
# `--no-project` skips monorepo resolution entirely; `--with-requirements`
# supplies only what copybarista actually needs. A project dependency-group
# cannot serve this role -- `--group` is ignored under `--no-project` -- so this
# requirements file is the unified place to add deps should copybarista ever
# need more than the one below.
#
# Consumed by scripts/sync_import_change.py and the sagent sync workflow.
pyyaml
